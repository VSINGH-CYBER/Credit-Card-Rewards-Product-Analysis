# Credit Card Rewards Transparency – Product Analysis Case Study

## Overview

This project analyzes a commonly found customer experience issue within credit card reward systems: **lack of transparency in reward calculations between transaction authorization and settlement.**

In many credit card platforms, customers do not see reward points until a transaction has fully settled. If the transaction amount changes (for example due to tips or merchant category adjustments), the final reward value may differ, more or less, from what customers expect. This can lead to confusion, lack of trust in reward calculation engine, and increased customer support inquiries.

This case study proposes a **product feature that displays an estimated (pending status) rewards value immediately after a purchase is authorized**, improving transparency and customer engagement.
